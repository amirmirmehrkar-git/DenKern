## 1. What the Temporal GNN Currently Outputs

The model is served via FastAPI (`inference/fastapi_wrapper.py`).
The export contract is in `output/temporal_gnn/frontend_inference_handoff.json`
(generated by `scripts/export_temporal_gnn_handoff.py`).

### POST /predict — Response Shape

```json
{
  "model_artifact": "output/temporal_gnn/temporal_gnn_model.pt",
  "timestamp": "<snapshot timestamp>",
  "num_ports_returned": 7,
  "num_vessels_returned": N,
  "queue_depth_predictions": [
    { "port_name": "Hamburg", "predicted_queue_depth_hours": 14.3,
      "mean_mc": 14.1, "std_mc": 0.8, "confidence": 0.84 }
  ],
  "transit_time_predictions": [
    { "vessel_index": 0, "predicted_transit_hours": 72.4,
      "mean_mc": 72.1, "std_mc": 3.2, "confidence": 0.76 }
  ],
  "confidence_definition": "confidence = exp(-std_mc / (abs(mean_mc) + 1.0))"
}
```

### Confidence Score Definition

```
confidence = exp(−std_mc / (|mean_mc| + 1.0))
```

This is an **uncertainty-derived proxy in [0, 1]**, not a calibrated probability.
It is NOT directly equivalent to `PredictionOutput.delay.confidence_score`.

## 2. Output Classification

| Output field | Classification | Notes |
|---|---|---|
| `queue_depth_predictions[].predicted_queue_depth_hours` | **Direct model output** | Per study port |
| `queue_depth_predictions[].mean_mc` / `std_mc` | Direct model output | MC-dropout samples |
| `queue_depth_predictions[].confidence` | **Derived** | `exp(−std / (|mean| + 1))` |
| `transit_time_predictions[].predicted_transit_hours` | **Direct model output** | Per vessel in latest window |
| `transit_time_predictions[].mean_mc` / `std_mc` | Direct model output | MC-dropout samples |
| `transit_time_predictions[].confidence` | **Derived** | Same formula |
| `model_artifact` | **Unsupported as model_version** | File path, not a semantic version |
| `timestamp` | **Unsupported** | Snapshot timestamp, not inference wall-clock |
| ETA dates (baseline, expected, optimistic, pessimistic) | **Unsupported** | Not produced by model |
| `delay.expected_delay_days` | **Unsupported** | Requires baseline ETA + transit delta arithmetic |
| `delay.p_delay_over_3_days` | **Unsupported** | No CDF exposed — cannot derive from current output |
| Risk drivers | **Unsupported** | Grid node features are raw inputs, not classified risk events |

## 3. Gap Analysis Against DenkKern PredictionOutput Contract

Our contract (`packages/types/src/prediction.ts`):

```typescript
interface PredictionOutput {
  shipment_id: string;
  model_version: string;
  generated_at: string;
  eta: { baseline, expected, optimistic, pessimistic };
  delay: { expected_delay_days, p_delay_over_3_days, confidence_score };
  risk_drivers: Array<{ type, location, severity, estimated_impact_days }>;
}
```

### Field-by-field status

| DenkKern field | James currently provides | Status | Action |
|---|---|---|---|
| `shipment_id` | Not in model output | ❌ Missing | Adapter must inject from caller context |
| `model_version` | File path only via `/health` | ⚠️ Partial | Adapter calls `/health`, parses version |
| `generated_at` | Snapshot timestamp (not inference time) | ⚠️ Partial | Adapter uses `new Date().toISOString()` |
| `eta.baseline` | Not produced | ❌ Not supported | Requires AIS/ERP contracted arrival date |
| `eta.expected` | Not produced | ❌ Not supported | Derivable only if baseline known |
| `eta.optimistic` / `pessimistic` | Not produced | ❌ Not supported | No agreed ±k×std formula |
| `delay.expected_delay_days` | Not produced | ❌ Not supported | Needs `(predicted_transit_hours − baseline_hours) / 24` |
| `delay.p_delay_over_3_days` | Not produced | ❌ Not supported | Needs agreed CDF or Gaussian formula |
| `delay.confidence_score` | Per-vessel `confidence` available | ⚠️ Partial | Adapter must select + aggregate relevant vessel |
| `risk_drivers` | Grid wind/wave as raw features | ❌ Not supported | Raw feature vectors ≠ classified risk events |

**Summary: 0 fields fully supported, 3 partial/derivable, 7 not supported.**

## 4. normalizeMinimalPrediction() Compatibility

`normalizeMinimalPrediction()` expects:

```typescript
interface PredictionOutputMinimal {
  shipment_id: string;
  expected_delay_days: number;
  p_delay_over_3_days: number;
  confidence_score: number;
}
```

**Verdict: Not directly compatible with Temporal GNN output.**

The GNN emits graph-level predictions at port and vessel granularity.
`PredictionOutputMinimal` expects shipment-level scalars.
The mapping requires a dedicated shipment-context-aware adapter that:

1. Selects the relevant vessel from `transit_time_predictions[]` via a `shipment_id → vessel_index` mapping (not in GNN output — James must expose this).
2. Converts `predicted_transit_hours` to `expected_delay_days` using a known baseline transit duration (requires ERP context, not in GNN output).
3. Estimates `p_delay_over_3_days` from MC distribution — not possible without an agreed CDF formula.
4. Selects the destination-port confidence for `confidence_score`.

**`normalizeMinimalPrediction()` must not be called directly on GNN `/predict` output.**
A dedicated `GNNShipmentAdapter` (future work) would produce `PredictionOutputMinimal`,
which `normalizeMinimalPrediction()` then upgrades to `PredictionOutput`. No downstream changes needed.

## 6. Minimum Adapter Spec (Future Work — Not Yet Implemented)

When James stabilises the contract, the integration path is:

```
POST /predict (GNN)
  → GNNShipmentAdapter.toMinimalPrediction(gnnResponse, erpContext)
    → PredictionOutputMinimal
      → normalizeMinimalPrediction(minimal, erpContext.required_by)
        → PredictionOutput   ← no downstream changes needed
```

### Blocking gaps James must close before integration:
1. **`shipment_id → vessel_index` mapping** — not in current GNN output.
2. **`p_delay_over_3_days` formula** — agreed Gaussian or empirical CDF from MC distribution.
3. **`model_version` string** — semantic version, not file path.

### What does NOT change when integration happens:
- `PredictionOutput` contract
- `normalizeMinimalPrediction()` function
- Scenario engine (pure function — no model awareness)
- All UI components

The adapter boundary absorbs the entire translation.
